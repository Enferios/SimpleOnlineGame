ТЕСТОВОЕ ЗАДАНИЕ


Оцениваться при проверке в первую очередь будут: архитектура, аккуратность кода, скорость выполнения задания.
Визуальная составляющая значения не имеет.
Проект может быть реализован полностью на Blueprints, но приветствуется и реализация на C++.

Необходимо сделать мини игру, цель которой, собрать достаточное количество нужных предметов.
Игрок появляется на карте, на территории которой есть несколько комнат. Между комнатами есть двери. Двери открываются по кнопке взаимодействия “E”.
Взаимодействие с предметами также осуществляется по кнопке “E”.

В игре есть телепорты, при нажатии на них (по кнопке взаимодействия “E”) игрока мгновенно переносит в другую точку. Можно сделать комнату, в которой нет дверей (или дверь заперта, т.е. с ней не проходит взаимодействие), но есть телепорт в соседнее помещение, например.

В комнатах случайно спавнятся три типа предметов (кубики, сферы и цилиндры, например).
Из них в начале каждой сессии случайно выбирается предмет, который нужно собирать для победы, т.е. либо мы собираем N количество кубиков, либо N количество сфер и тд (об этом необходимо сообщить в UI любым удобным образом).
С ненужными для победы предметами взаимодействие не проходит.
Также в UI необходимо выводить и количество собранных/количество нужных предметов (например Cubes: 2/5).

При выполнении условий победы выводить на экран надпись “YOU WIN” и через N количество времени начинать игру заново.

Время игровой сессии ограничено таймером на N секунд. Если таймер заканчивается, а условия победы ещё не выполнены - выводить на экран надпись “YOU LOSE” и через N количество времени начинать игру заново.
Таймер необходимо отображать в UI.

Весь код необходимо сделать с учётом мультиплеера. Не обязательно делать игру соревновательной, например можно сделать общие очки для всех игроков. Важно сделать адекватное отображение предметов и данных для всех игроков.
